## What to build

Dwa rodzaje dokumentacji, cross-cutting dla całego feature'a.

**(1) README** — dwie nowe sekcje:

- **"Sync z bankiem (opcjonalne)"** dla owner-ów gospodarstw. Kroki user-facing: kliknij "Połącz bank" → wybierz swój bank → zaloguj się na hosted page banku → wybierz konta → gotowe. Nic o GoCardless od strony usera — to wewnętrzny szczegół. Plus screenshots z dev środowiska pokazujące flow + jak wygląda inbox + badge `🏦`.

- **"Setup admina apki"** — admin-facing. Kroki: jak zarejestrować się u providera bank data, wygenerować klucze API, dodać przez `wrangler secret put` (`GOCARDLESS_SECRET_ID`, `GOCARDLESS_SECRET_KEY`), ustawić `ADMIN_EMAILS`. Wytłumaczenie limitów free tier (50 end-users, 90 dni historii). Co robić jak admin chce upgradować na płatny plan.

**(2) Polityka prywatności** — aktualizacja istniejącego dokumentu (jeśli jest) lub stworzenie nowego. Konkretnie:

- Wymienić GoCardless Ltd. jako data processor dla danych bankowych
- Wyspecyfikować jakie dane są transmitowane (IBAN, opisy transakcji, kwoty, daty)
- Retention: 90 dni w GC (limit free tier), w naszej DB tak długo jak user nie odłączy banku
- Podstawa prawna: zgoda usera + PSD2

**(3) Checkbox zgody przy pierwszym podpinaniu banku.** W flow Phase 2 (#16) na ekranie wyboru banku dodać checkbox "Akceptuję politykę prywatności [link] — wyrażam zgodę na przetwarzanie moich danych bankowych przez GoCardless Ltd. jako data processor". Zapamiętany per user (nie per household — bo jeden user może być owner kilku gospodarstw w przyszłości) w tabeli `users` jako kolumna `consent_bank_processor_at TIMESTAMPTZ NULL`. Bez zgody — przycisk "Połącz bank" nieaktywny.
