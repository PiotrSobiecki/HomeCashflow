## Type

**HITL** — acceptance wymaga real sync z GC sandbox + manualne kategoryzowanie w inboxie.

## What to build

Najgrubsza faza — wprowadza całą produkcyjną pętlę sync. Cloudflare Cron Trigger co 4h iteruje aktywne `bank_connections` z `next_sync_at <= NOW()` i ściąga nowe transakcje od `last_synced_at`. Każda transakcja mapowana po znaku kwoty (dodatnia → `kind=income`, ujemna → `kind=expense`, kwota brana jako `abs()`), trafia do `transaction_inbox` z `status=pending`.

Bez reguł kategoryzacji i bez dedupe vs manual — to robi Phase 4 (#TBD-4). Bez filtrowania transferów/ATM — też Phase 4.

Owner widzi nowy widok "Inbox" w apce z listą propozycji (opis, kwota, data). Dla każdej: dropdown kategorii (z `category_budgets`) + przycisk "Zatwierdź" → wpis ląduje w `transactions` z `source='bank'`, `created_by=owner.id`, `bank_inbox_id=inbox.id`. Drugi przycisk "Pomiń" → `status=dismissed`, nic w budżecie. Manual trigger `POST /api/bank/sync` (owner-only).

Zatwierdzone wpisy pokazują badge `🏦 z banku` w listach przychodów/wydatków. Edit kategorii i nazwy działa jak dla manualnych; PATCH z `amount` lub `txn_date` w body zwraca 400 z `field_frozen_bank` dla wpisów `source='bank'`.

Cały batch sync zapisany jako **jeden** wpis w `action_log` z `resource_type='bank_import'` i `after={count, connection_id, account_ids}`. Undo w 24h oknie cofa cały batch (usuwa wszystkie zatwierdzone wpisy z tego batcha + inbox rows).

Tabela `transaction_inbox` + kolumny `source` i `bank_inbox_id` w `transactions` powstają w tej fazie. Szczegóły w sekcji "Architectural decisions" i Phase 3 planu.

## Acceptance criteria

- [ ] Migracja drizzle dodaje `transaction_inbox` + kolumny `source` i `bank_inbox_id` w `transactions`
- [ ] Klient GC dodaje `getTransactions(accountId, dateFrom)` z testem
- [ ] Sync engine: integration test 0/1/N transakcji, mapowanie znaku, encrypt description/amount przed zapisem do inbox, idempotencja po `gc_transaction_id`
- [ ] Cloudflare Cron Trigger zarejestrowany na 4h cycle
- [ ] `POST /api/bank/sync` — manualny trigger, owner-only, 503 jeśli inny sync w toku
- [ ] Endpointy inbox: `GET /api/inbox?status=pending`, `POST /api/inbox/:id/approve { category }`, `POST /api/inbox/:id/dismiss`
- [ ] Inbox UI: lista, dropdown kategorii, Zatwierdź/Pomiń
- [ ] Badge `🏦 z banku` w `IncomeSection`/`ExpenseSection` przy `source='bank'`
- [ ] PATCH transactions: dla `source='bank'` blokuje zmianę `amount` i `txn_date` — test + 400 z `field_frozen_bank`
- [ ] `action_log` integration: 1 wpis per sync z `count: N`, undo cofa cały batch — integration test
- [ ] Manual E2E: sync z prawdziwego konta sandbox, widać w inboxie, zatwierdzasz, widać w budżecie z badge
